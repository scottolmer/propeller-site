The low mobile scores were reproducible, but the site was not CPU-heavy. The dominant cost came from render-blocking third parties, text intentionally hidden until JavaScript, font swapping, and homepage API work competing with first paint.

Root causes

Priority-ordered findings

1

Google Analytics occupied the critical path

The shared head loaded about 167 KB of Google tag JavaScript immediately, much of it unused during initial render. Blocking it alone moved analyzer performance to 90–91. The fix queues dataLayer events immediately and loads the remote library on interaction or after load during idle time.

2

The complete Lucide CDN runtime blocked parsing

Roughly 73 KB compressed was loaded synchronously to render a finite icon set. It is replaced by an 11 KB local subset that retains lucide.createIcons() compatibility and uses deferred execution.

3

Hero copy was invisible until JavaScript observed it

The LCP element on major templates had opacity:0 through the shared fade-in system. A critical inline rule now renders hero text immediately; below-the-fold animation behavior remains.

4

Font swapping caused repeatable layout shift

Analyzer, calculator, and picks desktop CLS clustered near 0.10–0.125. Shared fonts now use font-display: optional, eliminating the measured post-fix shift while retaining the branded face when it arrives within the optional window or is cached.

5

Homepage enrichment started during critical render

The static record was already in HTML, but JavaScript immediately initiated three API calls; one endpoint produced multi-second server latency. Hydration now waits for window load and idle time and retains the dated static snapshot if the API is slow or unavailable.

Guardrails

What prevents regression

The repository now includes a repeatable 5-page × 2-profile matrix runner, a Lighthouse CI configuration with performance and Web Vitals thresholds, a deterministic delivery optimizer, and unit tests that fail if an indexable page reintroduces blocking Google Analytics, the Lucide CDN, JavaScript-hidden hero copy, font swapping, or eager homepage hydration.

Measurement tradeoff: immediate analytics API loading was deliberately deprioritized behind the critical render. Events are queued before the library arrives, but a very fast bounce can be missed. This is an explicit product/measurement tradeoff, not a free optimization.
Field-data limitation: PageSpeed Insights and CrUX requests returned quota HTTP 429. No field Core Web Vitals pass is claimed. Confirm 75th-percentile LCP, INP, and CLS in Search Console or CrUX when data is available.
Remaining lower-priority opportunities

Not required to solve the measured defect

Additional image width variants, CSS splitting, and a CDN with longer immutable caching could improve repeat visits or tail latency. They were not the primary cold-load cause and should be evaluated against maintenance cost after production medians and field data are observed.
